# Glossary

## Goal

A measurable outcome the team wants to achieve.

## Team Goal

A measurable outcome shared by the team.

## Individual Goal

A measurable outcome assigned to one team member that supports a team goal.

## Owner

The team member responsible for a goal.

## Metric

The numeric or observable value used to track progress toward a goal.

## Target

The desired value or state that marks a goal as successful.

## Progress

The current measured movement toward a goal's target.

## Status

The current state of a goal, such as active, completed, or at risk.

## At Risk

A goal that may not be completed successfully because of low progress, a missed deadline, or a blocker.

## Blocker

Something preventing progress on a goal.

## Dashboard

The main view that summarizes goals, progress, and status for the team.

## Team Board

The authenticated dashboard view that organizes team goals, individual goals, owners, progress, and QA metrics for one team.

## Create Goal

The workflow for adding a new team or individual goal.

## Tests Passing

A QA metric showing whether a test category is currently passing.

## Test Coverage

A QA metric showing how much of the relevant code is exercised by tests.

## Unit Test

A test focused on a small unit of code, such as a function or component.

## API Test

A test focused on an application programming interface or backend endpoint.

## UI Test

A test focused on user interface behavior.
